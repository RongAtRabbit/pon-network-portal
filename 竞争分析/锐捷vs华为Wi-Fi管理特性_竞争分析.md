# 锐捷 vs 华为 Wi-Fi 管理特性竞争分析

## 1. 产品定位对比

| 特性 | 锐捷 N18E-XB/HCA | 华为 MA5800G |
|------|-------------------|-------------|
| **Wi-Fi管理方式** | 交换机作为AP上行汇聚，AP独立管理 | OLT集成Wi-Fi管理（CAPWAP） |
| **AP接入方式** | 以太网上行（GE/10GE） | PON光纤上行（GPON/XGS-PON） |
| **超融合方案** | Super Fiber (XGS-PON+Wi-Fi) | 无对应方案 |
| **云管平台** | WIS企业云 / MACC移动运维 | eSight |

## 2. 华为Wi-Fi管理特性（来源：华为Wi-Fi管理特性专题 02）

### 2.1 部署方式
- **eSight部署**：基于OLT端口，支持设备监控和故障预防
- **CLI自动部署**：基于ONT型号，OLT基于ONT型号配置策略
- **CLI手动部署**：分为ONT配置和WLAN配置两阶段

| CAPWAP服务 | OLT与AP之间建立CAPWAP隧道 |

| 负载均衡 | Wi-Fi负载均衡 |
| 智能漫游 | AP间客户端平滑切换 |
| AP信息查询 | 查询AP运行状态 |

## 4. 锐捷控标点

### ⚡ 锐捷必杀点

| 控标点 | 锐捷优势 | 华为对比 |
|--------|----------|---------|
| **VSU+AP跨设备冗余** | ✅ 支持AP跨VSU堆叠 | ❌ 不支持VSU，AP无法跨设备冗余 |
| **超融合方案** | ✅ Super Fiber (XGS-PON+Wi-Fi一体化) | ❌ 无对应方案，需独立采购AP+网关 |
| **WIS企业云** | ✅ 完整Wi-Fi云管理 | ⚠️ eSight功能有限 |
| **MACC移动运维** | ✅ 移动运维APP | ❌ 无移动运维方案 |
| **GRPC开放接口** | ✅ 支持GRPC/YANG | ❌ 不支持GRPC |
| **PoE++供电** | ✅ N18E-HCA/XBB支持90W PoE++ | ❌ 华为OLT不支持PoE |
| **终端容量** | ✅ 95000终端(带HCA) | ❌ 有限 |

---

## 5. 总结

**锐捷Wi-Fi方案优势：**
1. **VSU堆叠+AP跨设备**：锐捷核心优势，VSU堆叠环境下AP可实现跨设备冗余备份
2. **Super Fiber超融合**：在S7610平台上实现XGS-PON和Wi-Fi一体化，降低建网成本
3. **WIS+MACC云管**：完整Wi-Fi云管理+移动运维生态
4. **GRPC/YANG开放接口**：更开放的第三方集成能力

**华为Wi-Fi方案优势：**
1. **OLT集成CAPWAP**：AP直接通过PON光纤连接到OLT，简化布线
2. **eSight统一管理**：与光接入统一管理平台
