### 修复记录 ✅ 文档上传下载功能 - 服务器端存储

**问题:**
用户要求文档上传后存储在服务器中，而不是浏览器 localStorage。

**解决方案:**

**1. 创建文件上传服务器** (`file-server.py`)
- 基于 Python HTTP Server 的后端 API
- 支持文档上传、列表获取、下载、删除
- 文件存储在 `uploads/` 目录
- 元数据保存在 `uploads/docs.json`
- 最大文件大小: 50MB
- 支持格式: PDF, Word, PPT, Excel, CHM, TXT, MD

**API 接口:**
- `POST /api/upload` - 上传文件
- `GET /api/docs` - 获取文档列表
- `GET /api/download/<id>` - 下载文件
- `DELETE /api/docs/<id>` - 删除文档

**2. 修改前端代码** (`js/doc-server.js`)
- 使用 fetch API 与后端通信
- 使用 FormData 上传文件
- 支持上传进度显示
- 完善错误处理

**3. 更新文档中心页面** (`pages/docs/center.html`)
- 移除了 localStorage 存储逻辑
- 引用了新的 `doc-server.js`
- 更新提示文本为"文件存储在服务器中"

**4. 创建启动脚本**
- `start-all.sh` - 同时启动网站服务器和文件服务器
- `stop-servers.sh` - 停止所有服务

### 使用方法

**启动所有服务:**
```bash
./start-all.sh
```

**访问地址:**
- 网站: http://192.168.100.124:8080
- 文件API: http://192.168.100.124:8888

**停止服务:**
```bash
./stop-servers.sh
```
